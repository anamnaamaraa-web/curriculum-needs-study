# Стратегийн магистрын хөтөлбөрийн хэрэгцээний судалгааны систем

`Ажлын хэсгийн удирдамж (1).docx`-д заасан Макро–Мезо–Микро түвшний
судалгааг 4 үе шат, 18 ажлаар удирдах responsive front-end prototype.

## Ажиллуулах

`curriculum-needs-study` хавтсанд Node.js серверийг ажиллуулна:

```powershell
node server.mjs
```

Дараа нь `http://127.0.0.1:4173` хаягаар орно.
Үнэгүй локал шинжилгээ нь API түлхүүр, кредитгүй ажиллана. OpenAI гүн
шинжилгээг ашиглах бол workspace-ийн `.env.local` файлд `OPENAI_API_KEY`
тохируулж, API кредиттэй байна.

## Одоогийн боломж

- Удирдамжийн 4 үе шат, 18 ажлын төлөв, хариуцагч, нотолгооны хяналт
- PDF/TXT бодлого, стратегийн баримтаас текст задлах
- Файлын текстээс нэр, байгууллага, он, төрөл, чиглэл, URL, дугаарыг нотолгоотой дүрмээр автоматаар нөхөх
- Хөтөлбөрийн нэр, түвшин, зорилтот бүлгээр чадамжийн хамаарлыг сонгож эрэмбэлэх
- Чадамж бүрд бодлогын баримт бичгээс авсан нотлох заалтыг заавал харуулах
- Нэгдсэн чадамжийг хамаарах байгууллагаар шүүж харах, шүүсэн CSV татах
- Шинэ сургалтын хөтөлбөрийн PDF/TXT эсвэл текстийг шаардлагатай чадамжтай тулгаж, тусгасан/хэсэгчилсэн/илрээгүй төлөв ба эх заалт гаргах
- API кредитгүй, төхөөрөмж дээр ажиллах ил тод толь бичиг–эх өгүүлбэрийн локал шинжилгээ
- Ойлголт, хийх үйлдэл, суралцах үр дүн, үгүйсгэлийг нэг бүтэн өгүүлбэрийн хүрээнд хамтатган үнэлэх
- OpenAI Responses API-аар чадамж, үндэслэл, шууд ишлэл, суралцахуйн үр дүн гаргах
- JSON Schema-д баригдсан AI саналыг судлаачаар баталгаажуулах
- Олон баримтад давтагдсан чадамжийг нэгтгэн эрэмбэлэх, CSV экспортлох
- Албан ёсны/хянан магадалсан баримт бичгийн бүртгэл
- Фокус бүлгийн ярилцлагын үр дүнгийн бүртгэл
- Олон улсын хөтөлбөрийн харьцуулалтын матрицын өгөгдөл
- 8 стратегийн чадамжийн өнөөгийн түвшин–чухалчлалын асуулга
- Макро–Мезо–Микро нэгтгэл, skill-gap dashboard
- Тайлангийн 7 гаралтын бэлэн байдлын хяналт
- CSV болон төслийн JSON экспорт
- Responsive, keyboard-friendly интерфэйс

## Хязгаарлалт

Prototype нь `SUPABASE_URL` болон `SUPABASE_SECRET_KEY` тохируулсан үед мэдээллийг Supabase PostgreSQL-д хамтран хадгална. Supabase тохируулаагүй үед серверийн `data/shared-state.json` файлд хадгалж, browser-ийн `localStorage`-ийг түр fallback cache байдлаар ашиглана.
Үнэгүй локал горим нь бүтэн өгүүлбэрийн ойлголт, хийх үйлдэл, суралцах
үр дүн, үгүйсгэлийг ил тод дүрмээр хамтатган үнэлдэг; ерөнхий AI семантик
загвар биш тул үр дүн бүрийг судлаач эх заалттай тулган батална.
OpenAI гүн шинжилгээ нь API кредит/квот шаарддаг бөгөөд API түлхүүр зөвхөн
локал серверт уншигдана.
Бодит байгууллагын судалгаанд нэвтрүүлэхээс өмнө:

1. асуулгын агуулгын эксперт үнэлгээ, пилот туршилт хийх;
2. зорилтот олонлог, түүврийн хүрээ, урилгын болон сануулгын горим тогтоох;
3. сервер, өгөгдлийн сан, нэвтрэх эрх, backup болон audit log нэмэх;
4. мэдээлэл хамгаалал, хадгалах хугацаа, устгах журмыг батлах;
5. бүлгийн тайланд хамгийн бага хариултын босго тогтоох шаардлагатай.
